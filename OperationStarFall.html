Feature #1: Hitshader

On this project I have been buisy working with a hit-shader for when you get hit by a enemy or friendly fire. I made the beginning of the shader and later finished it together with an other developer

After the shader was done, I worked with the code, for when the player gets hit. It worked, but when we applied it to another player, the hitshader will be on at the same time if you get hit. So to fix that we had to make a copy of that shader to then later activate it to the applied gameobject.

When that was done, the other developer was fixing the bugs, while I was working with someone else working on the attacking drone. We fixed the bug that the drone could give damage to the player, then we fixed the knockback to the player for when it gets hit. After that was done we finally could merge that into each other with the hit-shader.

----------------------------------------------------------------

Feature #2: Unity Custom Editor

I am also working on a feature that gives unity a custom menu that can spawn in the prefabs what the others had made and can spawn in.

At first I made the layout of the custom menu and made labels for categories, then made scriptable objects with the values of those categories, like a string for which category it can be sorted in and then a list of prefabs that can be selected inside of the menu.

On the left of the text is the code of the first layout of the menu, for the categories, when I click on one of these categories, it will go to that category and put a bool to true, changing the layout of the menu.

On the right of the text is the code where the values of the categories will be put inside of the buttons in the custom menu, if you press one of these buttons you can spawn inside of the scene the prefab you selected.

I still have something planned for the custom menu where you can add a new prefab to a category and to add more categories inside of the custom editor, but that is going to be done when the first feature of the menu is done.
